# Промпт за нов A1 тест (целият процес в един чат)

Копирай блока между двата `===` реда в нов Sonnet 4.6 Thinking чат.
Смени **САМО** двете маркирани полета:

- `<<<ПАПКА>>>` — пълен път до папката с материалите.
- `<<<УРОЦИ>>>` — суфиксът на папката с теста, напр. `11` или `9-10`.

---

```
=========================================================================
ЗАДАЧА: Нов A1 тест — целият workflow в един чат.

ИЗТОЧНИК:  <<<ПАПКА>>>
ПОКРИВА:   уроци <<<УРОЦИ>>>
ПАПКА:     src/content/a1/tests/test-lessons-<<<УРОЦИ>>>/
ЕТАЛОН:    src/content/a1/tests/test-lessons-7-8/  ← използвай го като модел

─────────────────────────────────────────────────────────────────────────
ФАЗА 1 — ПЛАН (СПРИ за одобрение преди да пишеш каквото и да е)
─────────────────────────────────────────────────────────────────────────

1. Прегледай папката с материалите:
   • Ключ + текст за слушане (`kliuch-...txt`) → точки, отговори, „ТЕСТ А1.X".
   • Сканирани страници и картинките в `extracted-exercises -images/`.
   • Ако намериш `IMAGE-MAPPING.md` → пренебрегни го (често е остарял).

2. Прегледай ЕТАЛОНА (test-lessons-7-8) за структура, типове упражнения
   и формулировки. Прочети и `.cursor/rules/tts-audio.mdc`.

3. Представи план: id, totalPoints, секции с точки, картинки → подпапки,
   типове упражнения по секция, рискове. СПРИ и попитай за одобрение.

─────────────────────────────────────────────────────────────────────────
ФАЗА 2 — ИМПЛЕМЕНТАЦИЯ + TTS (само след одобрение, без прекъсване)
─────────────────────────────────────────────────────────────────────────

4. Копирай картинките в `public/assets/test-a1-X/{slushane,chetene,gramatika,pisane}/`
   с описателни kebab-case имена.

5. Създай `metadata.ts`, `exercises.ts`, `index.ts` (БЕЗ `content.ts`).
   Регистрирай теста в `src/content/a1/index.ts` (loaders + folder map).

   КЛЮЧОВИ ПРАВИЛА (всичко друго копирай от еталона):
   • Слушане: `reading_text` с `hideText: true` + true_false БЕЗ `title`
     (за да се слепят в едно упражнение).
   • Четене: `reading_text` с `noTranslation: true`.
   • Граматика, формат „прил. + същ. → _______ + същ." — пълен формат с
     повторено съществително; примерът да НЕ е същото като първия въпрос.
   • Писане: картинката е `imageUrl` ДИРЕКТНО на упражнението — без
     отделно `reading_text` упр. над него.
   • `points` = броят на `blanks` в упражнението.
   • totalPoints на metadata = сумата от секциите = сумата от ключа.
   • Числа в TTS текст с думи („две хиляди и първа"), без `=`, без `/`.
   • Бг съдържание (думи, отговори) НИКОГА не се превежда.

6. Валидация:
     npm run content:lint -- --test test-a1-X
     npx tsc --noEmit 2>&1 | Select-String "test-lessons-<<<УРОЦИ>>>"

7. TTS (Gemini Pro, женски Achernar default):
     npm run tts:generate -- --test <<<УРОЦИ>>> --model gemini
     npm run check:audio   # задължително, трябва да е зелено

   Ако липсва `.env.local` или `service-account.json` → спри и кажи.

─────────────────────────────────────────────────────────────────────────
ФАЗА 3 — ОБОБЩЕНИЕ
─────────────────────────────────────────────────────────────────────────

8. Дай кратък списък на променените файлове (код / активи / аудио) и
   завърши с:
     „Тестът е готов и валидиран. Пиши „commit" или „push" когато си готов."

   НЕ commit-вай / push-вай без изрично искане.
   НЕ пипай чужди файлове извън test-a1-X / src/content/a1/index.ts.

─────────────────────────────────────────────────────────────────────────
WINDOWS / POWERSHELL
─────────────────────────────────────────────────────────────────────────

Раздели команди с `;` (не `&&`). За файлове ползвай `Copy-Item -LiteralPath`,
`Get-ChildItem -LiteralPath`. За филтър на изхода: `... | Select-String "..."`.

Започни от Фаза 1. След плана спри и изчакай одобрение.
=========================================================================
```

---

**Какво даваш:** път до папката + кои уроци покрива (например `11` за `test-lessons-11`).

**Какво НЕ е нужно:** текстове, имена на картинки, номер на теста — извличат се от ключа.

**Pre-flight:** `.env.local` + `service-account.json` в корена на проекта.

**Модел:** Sonnet 4.6 Thinking. Opus само ако възникне нов тип упражнение.
